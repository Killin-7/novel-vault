# Novel Vault · 小说仓库

> OpenHanako 插件 · 跨小说的结构化知识库  
> 给 AI 一个不遗忘的写作上下文。

Novel Vault 不是「从 0 开始一键全自动写小说」的工具。它不会替你构思完整故事、不会直接生成整本小说，也不会代替作者审稿。

它的定位更具体：**辅助 AI 写长篇连载时保持上下文连续性**。

AI 写长篇连载时，最常见的问题不是一句话写不好，而是**记不住**：记不住前面埋过什么伏笔，记不住角色关系推进到了哪一步，记不住已经确认过什么设定。每次新开一个对话，这些信息都要重新喂给 AI；喂不全，就容易写出前后矛盾、人物跳变、伏笔遗忘的问题。

Novel Vault 做的事情是：**把你在写作过程中积累的结构化信息——章节规划、剧情汇总、伏笔状态、角色关系弧、设定约束、写作规则——整理成 AI 当前续写最需要的上下文，供 Agent 查询和注入。**

它不替代你已有的 Markdown 文件体系。章节规划、剧情汇总、伏笔表、大纲、角色设定仍然由你维护，格式不变。插件负责在需要的时候，把分散在多个文件中的信息解析、关联、分层，提供给 AI；同时使用插件私有数据文件保存 AtomicFact、写作规则和解析缓存。

它能辅助你完成这些事：

- **写前查状态。** 在 AI 续写前查询当前章节进度、最近剧情、角色关系、活跃伏笔和大纲目标
- **归档章节。** 一章写完后，把「发生了什么 / 情绪变化 / 关系变化 / 伏笔推进 / 设定新增」写入固定追踪文件
- **防止伏笔遗忘。** 把活跃伏笔、已回收伏笔、最近推进章节整理出来，供 AI 写下一章前查看
- **保护角色连续性。** 查询角色状态弧、说话方式、情绪触发点和写作规则，减少人物突然变味
- **检测情感节奏。** 对比相邻章节的情绪变化，提示是否出现重复回环
- **沉淀写作规则。** 把修改中总结出的角色声口、文风约束存成可检索规则，后续续写时继续使用

它不能替你完成这些事：

- 从一个空白想法自动生成完整小说项目
- 自动判断一本小说「应该怎么写才好看」
- 代替作者审稿、取舍剧情和决定人物命运
- 在没有规范化 Markdown 文件的情况下凭空知道你的小说设定

更准确地说，Novel Vault 是一套 **AI 小说写作的上下文管理层**。作者负责创作判断，AI 负责生成和协作，Novel Vault 负责让 AI 在写第 N 章时，知道自己站在前 N-1 章的肩膀上。

## 快速开始

1. **安装插件** — 在 Hanako → 设置 → 插件中，拖入插件文件夹，开启「允许全权插件」
2. **配置路径** — 在 Hanako → 设置 → 插件 → Novel Vault 中，将 `novelBasePath` 填为你存放小说项目的根目录路径（比如 `D:\我的小说` 或 `/home/user/novels`）
3. **打开页面** — 重启 Hanako，顶部导航栏出现「小说仓库」tab，点击即可浏览所有小说项目
4. **确保项目符合格式** — 每个小说文件夹需包含 `章节规划.md` 和 `正文/` 才会被识别。完整格式说明见 [`docs/NOVEL-PROJECT-SPEC.md`](docs/NOVEL-PROJECT-SPEC.md)

> **零依赖**：纯 JS，无 `npm install`，无原生模块。

## 典型使用方式

Novel Vault 更适合嵌入到已有的 AI 写作流程中，而不是单独承担全部创作工作。

一个常见流程是：

1. **作者维护项目文件**：在 Markdown 中维护章节规划、剧情汇总、伏笔表、角色设定和大纲
2. **写前查询上下文**：让 Agent 调用 `query-novel-state`，获取当前章节之前的剧情、关系、伏笔、设定和写作规则
3. **AI 续写章节**：AI 根据作者给出的目标和 Novel Vault 提供的上下文辅助写作
4. **作者审稿修改**：作者决定哪些内容保留、哪些内容重写，必要时总结新的角色声口或文风规则
5. **归档章节信息**：调用 `archive-chapter` 写入剧情汇总、章节规划和伏笔状态；必要时补充 AtomicFact
6. **下一章继续复用**：后续写作时，AI 可以继续查询这些已沉淀的信息

换句话说，它解决的是「AI 写长篇时如何不忘前文」的问题，而不是「如何自动从零生成一本小说」的问题。

## 功能

- **小说概览** — 所有小说项目的进度一览（章节数、已归档数、未解决伏笔数）
- **章节看板** — 每章详情 + 写前必读（角色状态弧 × 大纲对照 × 章末钩子预览）
- **伏笔追踪** — 伏笔看板（活跃 vs 已回收）、编号关联检索
- **角色时间线** — 基于大纲的情绪状态序列
- **Agent 工具** — 在对话中直接调用的 AI 辅助工具：
  - `archive-chapter` — 归档章节，自动写入剧情汇总、更新规划表、维护伏笔状态，支持 AtomicFact 细粒度事实
  - `query-novel-state` — 查询小说完整状态（full 模式）或续写简报（brief 模式，L0/L1/L2 三层），支持写作规则查询
  - `detect-emotion-pattern` — 情感节奏检测，避免连续章节的情绪回环
  - `search-facts` — 按角色/关键词检索已归档章节的细粒度事实
  - `refine-skill` — 存入写作规则，持久化角色声口和文风约束
  - `query-skills` — 查询已存储的写作规则
- **主题自适应** — UI 使用 OpenHanako 原生 CSS 变量，自动跟随深浅主题切换

## Agent 工具

插件向 AI Agent 暴露六个工具，可在对话中直接调用。

调用工具时需要指定小说的 `slug`。`slug` 是小说文件夹名的拼音化转换（中文→拼音、空格→连字符），你可以在插件的 UI 页面中看到每部小说的 slug 值。

### archive-chapter — 归档章节

Agent 提供写好的 5 项记录，工具自动写入三个文件并 `.bak` 备份。

```json
{
  "slug": "my-novel",
  "chapter": 7,
  "entries": {
    "what": "主角第一天正式上班…",
    "emotion": "紧张→松动→确认感",
    "relation": "关系从观察式照顾进入确认式照顾",
    "foreshadow": "F06推进中；F10已埋下",
    "setting": "确认故事世界的一条设定规则"
  }
}
```

写入目标：`剧情汇总.md`（追加）、`章节规划.md`（更新计数+状态表）、`伏笔与回收表.md`（更新状态列）。支持可选的 `atomic_facts` 字段，归档时同步产出可独立检索的细粒度事实。

### query-novel-state — 查询小说状态

```json
{ "slug": "my-novel", "mode": "brief", "character": "主角名" }
```

- `mode: "full"` 返回全量数据
- `mode: "brief"` 返回续写简报（L0 核心上下文 / L1 章号关联 / L2 深查指针）
- 传入 `character` 参数可获取该角色的写作规则（writing_skills）

### detect-emotion-pattern — 情感节奏检测

对比相邻章节的情绪词变化，检测回环模式（放下→不安→放下…）并提示节奏问题。词库覆盖 50+ 微妙情绪词。

```json
{ "slug": "my-novel", "recentChapters": 3 }
```

### search-facts — 细粒度事实检索

按角色名或关键词检索已归档章节中的原子事实。Agent 在续写涉及特定角色的场景时调用。

```json
{ "slug": "my-novel", "query": "角色名,关键词" }
```

### refine-skill — 存入写作规则

Agent 在修改角色台词或文风后，将提炼出的写作规则持久化。

```json
{ "slug": "my-novel", "character_name": "主角", "scope": "内心独白", "rule": "内心独白最多两句" }
```

### query-skills — 查询写作规则

按角色名和适用范围查询已持久化的写作规则，按置信度降序返回。

```json
{ "slug": "my-novel", "character_name": "主角", "scope": "对话" }
```

## 常见问题

### 填了路径但页面不显示任何小说

检查你的小说项目根目录下的子文件夹是否同时满足两个条件：
- 包含 `章节规划.md` 文件
- 包含 `正文/` 目录

只有同时满足这两项，文件夹才会被识别为小说项目。

### 调用工具时提示「novelBasePath 未配置」

在 Hanako → 设置 → 插件 → Novel Vault 中找到 `novelBasePath` 配置项，填入你的小说项目根目录路径。

### 页面切换视图后白屏

当前版本的视图切换依赖 JavaScript DOM 路由。如果出现白屏，刷新页面即可恢复。

### 插件和我的现有写作流程怎么配合

插件会读取你已有的 Markdown 文件；默认不会修改小说项目文件，除非你调用 `archive-chapter` 工具，它会在写入前自动 `.bak` 备份。`refine-skill` 与 AtomicFact 会写入插件自己的私有数据文件，不会改动你的小说正文或设定文档。你可以继续用任何编辑器写小说，插件负责提供查询、归档和持久化记忆能力。
